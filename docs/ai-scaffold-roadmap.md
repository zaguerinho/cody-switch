# AI Scaffold Roadmap

Master plan for evolving claude-switch from a feature context manager into a full project scaffold for AI-assisted development.

## Vision

One command (`claude-switch init`) makes any repo AI-ready. Directory-specific contexts, testing guidelines, and stack-aware templates give Claude focused, relevant instructions at every level.

## Milestones

### 1. `claude-switch init` — Project Bootstrap
**Status:** ✅ Complete
**Priority:** Highest — anchor command everything else builds on

- [x] Tech stack detection (package.json, pyproject.toml, go.mod, Cargo.toml, Makefile, etc.)
- [x] Test framework detection (jest, pytest, go test, bats, etc.)
- [x] Seed project CLAUDE.md with detected stack + testing sections
- [x] Create `.claude/features/` structure
- [x] Auto-add `.gitignore` entries for claude-switch artifacts
- [x] `--template` override flag (node, python, go, bash, generic)
- [x] Skip/warn if project already initialized
- [x] `--force` flag for reinitialize with backup
- [x] Build tool / package manager detection (npm/yarn/pnpm, pip/poetry/uv, etc.)
- [x] Tests in `test/init.bats` (13 tests)

### 4. `claude-switch status` — Health Dashboard
**Status:** ⏳ Pending
**Priority:** Medium

Show what's configured vs recommended:
- `.claude/features/` exists
- Root CLAUDE.md active
- `.gitignore` entries present
- Directory-specific contexts (suggest candidates)
- Testing guidelines present
- Feature/lesson counts

### 5. Directory-Specific CLAUDE.md Management
**Status:** ⏳ Pending
**Priority:** Medium

- `claude-switch dir-context <path>` creates `.claude/CLAUDE.md` in target directory
- Starter template scoped to that directory's purpose
- `claude-switch dir-context --list` shows all directory contexts
- Claude Code reads these natively — tooling just makes creation/management easy
